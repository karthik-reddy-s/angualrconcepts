1.constructor-every class has it,firstexecutes,used for dependency injection
2.ngonchanges-when there is a value of input control withina comp has been changed
3.ngonInit-this first initializeswhen comp has been intialized,it will come after ngonchanges event
4.ngdocheck-everytime the input properties of comp are checked.
5.ngaftercontentinit-is executed when angular performs any content projection with component views.
6.ngaftercontentchecked-everytime content of comp has been checked by change detection mechansim of angular
7.ngafterviewinit-when the comp view has been fully initalized.
8.ngafterviewchecked-everytime the view of given comp has been checked by change detection algo of angular
9.ngondestroy-before destroy the component.
order is constructor,ngonchanges,ngoninit,ngdocheck,ngaftercontentinit,ngafterviewinit ngaftercontentchecked,ngafterviewinit,ngafterviewchecked. later we get ngdocheck,ngaftercontentinit,ngafterviewinit.
if we write or remove something we get,ngdocheck,ngaftercontentchecked,ngafterviewchecked.
